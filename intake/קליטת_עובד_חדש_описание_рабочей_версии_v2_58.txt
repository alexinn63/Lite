# קליטת עובד חדש — описание рабочей версии и порядок восстановления

Дата фиксации: 22.07.2026  
Текущая рабочая версия: `קליטת_עובד_חדש_v2_58_finish_step_fix.zip`

## 1. Назначение системы

Система автоматизирует приём нового получателя услуг / работника:

1. Получает комплект документов.
2. Распознаёт имя, фамилию и ת.ז.
3. Определяет тип каждого документа.
4. Позволяет проверить и исправить результаты.
5. Создаёт папку человека и стандартные подпапки.
6. Раскладывает документы по нужным папкам.
7. Обновляет `תיק דיגיטלי.xlsx`.
8. Обновляет банковский Excel.
9. Открывает Outlook с банковским документом.
10. Копирует готовый текст письма в буфер обмена.
11. Переводит систему на этап `6. סיום`.

---

## 2. Главный файл программы

Основной файл:

`index.html`

Он работает локально в Chrome или Edge. Интернет-сервер не требуется.

Запуск:

1. Распаковать ZIP в обычную папку.
2. Не запускать `index.html` прямо из окна WinRAR.
3. Открыть распакованную папку.
4. Дважды нажать `index.html`.

## 4. Этапы интерфейса

1. `קבלת מסמכים` — получение документов.
2. `זיהוי אדם` — определение человека.
3. `בדיקת מסמכים` — проверка и классификация документов.
4. `יצירת תיק` — создание папки и распределение файлов.
5. `עדכון מערכות` — обновление Excel и открытие Outlook.
6. `סיום` — завершение процесса.

После успешного открытия Outlook пункт 6 должен стать активным/зелёным.

---

## 5. Распознавание человека

Обязательные поля:

- имя;
- фамилия;
- ת.ז.

Главный источник имени и фамилии — содержание документов, особенно явно подписанные поля:

- `שם פרטי`
- `שם משפחה`
- `מספר זהות`
- `תעודת זהות`

Название файла не должно быть главным источником имени. Оно может использоваться только как слабая подсказка.

Если имя, фамилия или ת.ז. определены неправильно, пользователь должен иметь возможность исправить их вручную.

Ручные исправления нельзя автоматически перезаписывать результатами OCR.

---

## 6. Распознаваемые документы

Основные типы:

- תעודת זהות / ספח
- מסמך בנק / אישור ניהול חשבון / צילום צ'ק
- ביטוח לאומי
- דוח רפואי / אישור רפואי
- מסמך ועדה / סל שיקום
- ויתור סודיות
- תוכנית שיקום
- מסמכי מדריך
- מסמכי אפוטרופוס
- מסמך כללי

Для PDF без текстового слоя используется OCR.

Для банковского документа должны определяться:

- название банка;
- номер банка;
- номер отделения;
- номер счёта.

Если номер счёта не определился, его можно ввести вручную. После сохранения введённые банковские данные должны появиться в строке документа.

## 7. Структура папки человека

Создаётся папка с именем человека.

Пример:

`אליעזר רן סויסה`

Внутри создаются стандартные папки:

- `כללי`
- `דוח רפואי`
- `הועדה`
- `ויתור סודיות`
- `מדריך`
- `עו_ס`
- `עו_ס/תיעוד שיחה`

Папка `אפוטרופוס` создаётся только при наличии соответствующего документа или явного указания пользователя.

Распределение:

- תעודת זהות / ספח — в корень папки человека;
- банковский документ — `כללי`;
- ביטוח לאומי — `כללי`;
- медицинские документы — `דוח רפואי`;
- ועדה / סל שיקום — `הועדה`;
- ויתור סודיות — `ויתור סודיות`;
- תוכנית שיקום — `עו_ס`;
- документы מדריך — `מדריך`;
- אפוטרופוס — `אפוטרופוס`.

---

## 8. Если папка человека уже существует

Процесс не должен останавливаться.

Используется существующая папка:

- существующая папка не удаляется;
- остальные находящиеся в ней документы не удаляются;
- новые документы добавляются;
- если новый документ имеет такое же имя, его разрешено перезаписать;
- после этого процесс продолжается к Excel.

Отдельная кнопка «Отмена» не нужна. Пользователь может закрыть окно программы.

---

## 9. Работа с Excel

Используются два файла:

### Главный реестр

`תיק דיגיטלי.xlsx`

Система:

- проверяет наличие человека по ת.ז.;
- не должна создавать дубль;
- добавляет новую строку;
- заполняет имя, фамилию, ת.ז., מדריך и даты;
- создаёт относительные ссылки на папки и документы;
- сохраняет существующее форматирование.

### Банковский реестр

`מספרי חשבון מקבלי שירות גבע.xlsx`

Лист:

`גיליון1`

Заполняются:

- ת.ז.;
- фамилия;
- имя;
- банк;
- номер банка;
- отделение;
- номер счёта.

Поле SAP остаётся пустым, если номера нет.

### Важные правила Excel

- Excel должен быть закрыт во время обновления.
- Файл не должен быть защищён паролем на открытие.
- Сейчас резервная копия автоматически не создаётся.
- После записи должно появиться сообщение, что данные скопированы в выбранный файл.
- Используются относительные ссылки, а не абсолютные пути вида `F:\...`.
- Не менять `hyperlinkBase` внутри Excel XML.

## 10. Письмо Сиван

Адрес:

`sivans@yachdav.org.il`

Финальная кнопка:

`סיום ופתיחת מקבל שירות חדש`

После нажатия:

1. банковский документ открывается через окно `Share using Outlook`;
2. готовый текст письма копируется в буфер обмена;
3. пользователь выбирает или вводит адрес Сиван;
4. ставит курсор в тело письма;
5. нажимает `Ctrl+V`;
6. проверяет письмо и отправляет;
7. система сразу переходит на `6. סיום`.

Окно Outlook программа не закрывает.

Пример текста письма:

נושא: מקבל שירות חדש – [שם מלא]

שלום סיון,

יש לי מקבל שירות חדש:

ת.ז.: [מספר זהות]
שם משפחה: [שם משפחה]
שם פרטי: [שם פרטי]
בנק: [שם הבנק]
מספר בנק: [מספר בנק]
מספר סניף: [מספר סניף]
מספר חשבון: [מספר חשבון]

מצורף אישור חשבון הבנק.

בברכה,
אלכס

Ограничение Windows Share:

- вложение передаётся автоматически;
- получатель, тема и текст могут не передаваться;
- поэтому текст заранее копируется и вставляется через `Ctrl+V`.

Не использовать вариант `.eml`: он создаёт лишний файл и требует дополнительного открытия.

## 11. Завершение процесса

После открытия Outlook система должна:

- активировать пункт `6. סיום`;
- показать сообщение о завершении;
- оставить кнопку для начала новой קליטה.

Ожидаемый текст:

`תהליך הקליטה הושלם. המייל לסיון נפתח ומוכן להשלמה ושליחה.`

Кнопка:

`התחלת קליטה חדשה`

## 12. Безопасность

1. Не удалять оригинальные документы автоматически.
2. Не удалять существующую папку человека.
3. Проверять дубль по ת.ז.
4. Не добавлять второй раз человека в Excel.
5. Банковские данные сохранять только после проверки пользователем.
6. Не перезаписывать ручные исправления OCR.
7. Не использовать защищённый паролем Excel без отдельной доработки.
8. Перед крупными изменениями программы сохранять копию последней рабочей версии.
9. Каждую новую версию проверять на одном человеке полностью от начала до конца.
10. Не менять уже работающие части одновременно с новой функцией.

7. Проверить банковскую строку.
